# 잔류 - 개발 전달용 요약본

## 게임 개요
- 폭우로 침수된 지하철역에 고립된 승객이, 점점 기괴해지는 안내와 규칙 속에서 살아남아야 하는 2D 픽셀 공포 탐험 게임

## 핵심 방향
- 전투 없음
- 도망, 회피, 탐색, 관찰 중심
- 사운드보다 전광판형 안내와 짧은 나레이션 중심
- 공포는 점프스케어보다 위화감, 반복, 공간 압박, 추적 중심
- 물은 수위 상승보다 공간 잠식으로 표현

## 구역 구조
- 0구역: 역사 입구 / 개찰구 — 튜토리얼, 현실 기준점
- 1구역: 대합실 — 이상한 안내와 첫 위화감
- 2구역: 승강장 — 물 잠식과 첫 추적
- 3구역: 환승 통로 / 하부 통로 — 반복 구조, 잘못된 표지판, 개인화된 개입
- 4구역: 폐쇄 선로 아래 — 분류 시설의 정체와 마지막 판정

## 분기 구조
- 마지막 선택 하나보다, 플레이어가 역을 어떻게 대했는지에 따라 분기
- 핵심 분기 축
  - 규칙 준수
  - 오염
  - 조사도
  - 유인 반응

## 엔딩 구조
- 탈출: 살아 나왔지만 완전히 벗어나지 못한 느낌
- 루프: 다시 처음으로 돌아옴
- 편입: 잔류 승객이 됨
- 오인 구조: 구조처럼 보였지만 사실은 아님
- 침묵: 살아남았지만 일부가 비어 있음

## 연출 원칙
- 긴 설명보다 환경 연출 우선
- 메모, 문서, 반사, 전광판 문구로 정보를 조각내서 전달
- 같은 문장은 완전히 다르게 쓰기보다 미묘하게 어긋나게 반복
- 플레이어가 직접 이상함을 눈치채게 만들 것

## 이미지 프롬프트 설계 병행
- 개발과 함께 이미지 생성용 프롬프트 세트도 병행 정리
- 배경, 컷씬용 일러스트, 연출 화면, 오브젝트 레퍼런스용 프롬프트 포함
- 필요한 화면 단위를 먼저 목록화
- 각 장면마다 아래 요소 포함
  - 구도
  - 조명
  - 분위기
  - 화면 정보
  - 픽셀아트 톤
- 구현뿐 아니라 아트 리소스 제작 전달용 문서까지 고려

## 절대 바뀌면 안 되는 것
- 전투 중심으로 바꾸지 말 것
- 사운드 연출 의존형으로 바꾸지 말 것
- 긴 대사/긴 컷씬 위주로 풀지 말 것
- 단순 유령 추격전으로 축소하지 말 것
- 이 역이 재난 현장처럼만 보이게 하지 말 것

## 한 줄 정리
- 침수된 지하철역에서 잘못된 안내, 공간 잠식, 추적, 반복 속에 플레이어를 분류하는 공포 탐험 게임
